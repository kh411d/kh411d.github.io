Go extending Array with a Real-time Observer

Khalid Adisendjaja · March 13, 2021

There might be some circumstances where you’re going to need to monitor an array as it’s in real-time, whether it’s appended, done something to a value as it added, trigger an action, or anything else.

Luckily, in Go, we can create a custom type variable and benefit from the go-routine itself. For this tutorial, we’re going to have an array that has an observer to monitor its values.

var observee []int
observee = append(observee,1)

The idea, every time a value is added to the observed variable observee, the observer could immediately take action in real-time, as long as the main thread has not ended yet.

Rather than using the default type []int, let’s make a custom type called observee so later we can freely extend the type.

Furthermore, to make the observing happen in real-time we need to create the observer’s routine. In other words, the observer needs to have its lifespan and continuously receiving information.

type observee []int
func (o *observee) observer(fn func(observee)) {
    //Start a go routine
    go func() {
        var pastLen int
        //Loop forever
        for {
            newLen := len(*o)
            if newLen != pastLen {
                //Send only when there are changes
                fn(*o)
            }
            pastLen = newLen
        }
    }()
}
func main() {
  var theArray observee

  theArray.observer(func(o observee) {
     fmt.Println(o)
  })

  for i := 1; i <= 3; i++ {
      theArray = append(theArray, i)
      time.Sleep(1 * time.Second) //append the array per 1 second
  }
}

If we run the code above, the observer will print the array every time appended per second. However, running a forever loop without knowing how to stop is not a good idea. So, what if we create a more controllable observer? In which we can restart/kill the observer and modify the observe duration like a ticker.

type observee []int
func (o *observee) observer(fn func(observee), td time.Duration) *time.Ticker {
    ticker := time.NewTicker(td)
    //Start a go routine observer
    go func() {
        var pastLen int
        //Loop forever
        for {
            select {
                case <-ticker.C:
                newLen := len(*o)
                //Compare the length
                if newLen != pastLen {
                    //Do something!
                    fn(*o)
                }
                pastLen = newLen
            }
        }
    }()
    return ticker
}

func main() {
	var theArray observee

	t := theArray.observer(func(o observee) {
		fmt.Printf("Observe %v\n", o)

	for i := 1; i <= 3; i++ {
		theArray = append(theArray, i)

	fmt.Println("Ticker/Observer stopped")
	fmt.Println("Resume appending, but observer will skip 4,5,6")

    t.Reset(1 * time.Nanosecond)

	for i := 7; i <= 9; i++ {
		theArray = append(theArray, i)
		time.Sleep(1 * time.Second)

	fmt.Printf("The final array: %v\n", theArray)
}

type observee []int
func (o *observee) observer(fn func(observee), td time.Duration) chan struct{} {
    done := make(chan struct{})
    //Start a go routine
    go func() {
        var pastLen int 
        //Loop forever
        for {
            select {
            case <-done:
                return
            case <-time.NewTicker(td).C:
                nowLen := len(*o)
                if nowLen != pastLen {
                    //Do something!
                    fn(*o)
                }
                pastLen = nowLen
            }
        }
    }()
    return done
}

func main() {
  var theArray observee
  var wg sync.WaitGroup
  var mu sync.Mutex

  t := theArray.observer(func(o observee) {
      fmt.Printf("Observe %v\n", o)
  }, 1*time.Nanosecond)

  for i := 1; i <= 10; i++ {
      wg.Add(1)
      go func(w *sync.WaitGroup, i int) {
          defer wg.Done()
          //Make a random pause for each go routine to simulate process time before appending
          time.Sleep(time.Duration(rand.Intn(3)) * time.Second)
          //Append theArray with lock
          mu.Lock()
          theArray = append(theArray, i)
          mu.Unlock()
      }(&wg, i)
  }

  wg.Wait()
  //kill the observer
  t <- struct{}{}
  fmt.Printf("The final array: %v\n", theArray)
}

As expected, the value of the theArray will not be in order, this happens because all the routines in accessing theArray happen randomly, as you see that I’m using mu.Lock() and mu.Unlock() before and after appending theArray feel free to comment those lines and see what happened.
